What stays local

Meeting Note stores recordings, transcripts, summaries, logs, and its SQLite database locally under the app data folder. The app is not a hosted SaaS dashboard.

What leaves your computer

Audio is sent to OpenAI speech-to-text when you request transcription. Summary prompts are sent to your OpenRouter-compatible model provider. API usage depends on the keys and providers you configure.

Optional diagnostics

Anonymous product milestones and crash signals are off by default. If you opt in, the app sends the event name, app version, a random installation ID, and coarse technical properties to private Google Forms. It never includes recordings, transcripts, summaries, meeting titles, API keys, or logs.

Release builds may also use a Sentry-compatible service for scrubbed crash stack traces. Default personal data, local variables, source context, device name, paths, email addresses, credentials, and breadcrumbs are removed before delivery.

Updates and feedback

Automatic update checks contact GitHub Releases at most once per day and do not send the random installation ID. Feedback text and an optional reply contact are stored in a private Google Form only after you press Send.

Website measurement

This website uses Yandex Metrika to measure visits and download clicks. Metrika may set browser identifiers and process interaction data under Yandex's service terms.

Use it responsibly

Only record meetings when you have the right consent for your jurisdiction and organization. Meeting content can be sensitive even when it is stored locally.

For a private workflow, start with a test recording and review your app data folder.
